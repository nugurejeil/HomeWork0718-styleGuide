  • 들여쓰기

    들여쓰기는 2칸의 공백문자로 처리. Tab 방식보다 코드의 전체적인 모양을 파악하기 수월합니다.

  • HTML5 문서타입 선언

    문서 타입을 선언하여 하여 브라우저가 표준모드로 작동하도록 합니다. 

            
          <!DOCTYPE html>

  • 언어 속성

    언어의 속성을 설정하여 스크린리더기와 같은 도구들이 어떠한 발음을 사용해야 하는지, 어떠한 번역 도구를 사용해야 하는지 결정하도록 도와줍니다. 

            
          <html lang="ko-KR">

  • 문자열 인코딩 선언

    문서의 문자열을 보다 빠르고 정확하게 처리할 수 있도록 도와줍니다. 

            
          <meta charset="UTF-8">

  • 프라그마 디렉티브(pragma directive)

    IE의 가장 최신의 브라우저에서 사용하는 방식으로 문서를 렌더링 하도록 합니다. 

            
          <meta http-equiv="X-UA-Compatible" content="IE=Edge">

  • SEO

    문서와 관련된 내용을 상세하게 작성해서 서치엔진이 문서를 잘 찾을 수 있도록 도와줍니다. title 태그의 경우 브라우저 상단 탭에 보여짐으로 사용자가 페이지의 구분을 용이하게 하기 위해 중복된 내용을 사용하지 않습니다. 

            
          <title>My title</title>   
          <meta name="description" content="">
          <meta name="keywords" content="">

  • 모바일 최적화

    컨텐츠의 크기가 사용자 디바이스 화면의 비율에 알맞게 조절되도록 도와줍니다. 

            
          <meta name="viewport" content="width=device-width, initial-scale=1.0”>

  • 작성자 명시

    페이지를 작성한 사람이 누구인지 명시합니다. 오류를 낸 사람이 누군지 색출하여 처단하기 위해. 

            
          <meta name="author" content="">

CSS

  • CSS 속성 작성 우선순위

    layout에 가장 영향을 미치는 순서로 작성함을 원칙으로 합니다.

    1. display, float, position
    2. width, height, margin, padding
    3. border, font-size, font-weight
    4. color, background-color
    5. transform, transition, animation

  • CSS 클래스 네이밍 규칙

    1. html 페이지의 이름 (공통모듈인 경우 페이지 대신 ‘global’)
    2. html 문서 안의 모듈(header, main, footer…)
    3. 모듈안의 신텍스(혹은 기능, 쓰임새(nav, aside, img…)
    4. 구체적인 기능
            
          <div class="global-main-aside-advertise">

  • img 태그 속성 규칙

    alt
    이미지에 대한 설명이나 묘사. 이미지 안에 글이 들어가 있는 경우 글의 내용을, 이미지만 있으면 이미지에 대한 간략한 설명을 작성합니다.
    title
    이미지의 제목, 혹은 주제를 작성합니다.
            
        
          <img src="#" alt="Pokemon Go surprising Pikachu" title="Pokemon Go in game scene">

  • SMACSS 방법론을 적용한 모듈화

    코드의 구조를 알아보기 쉽게 하고 유지 보수하는데 유용합니다.

    BASE CSS
    html, body, header나 footer, clear fix 등의 요소에서 공통적으로 쓰이는 속성들(Reset.css 와 비슷하다고 보면 됩니다.)
    LAYOUT CSS
    레이아웃과 관련된 CSS(float, 박스모델에 관련된 속성들). 요소들을 어떻게 나열할지 결정합니다. 보통 ID셀렉터를 이용하는데 페이지에서의 중요도를 어필하기 위함입니다.
    MODULE CSS
    여러 페이지에서 반복적으로 재사용되는 것들이 이에 해당됩니다. 어디서든 사용되기 때문에 고유의 클래스 이름을 지정해줍니다.
    STATE CSS
    요소의 상태에 대한 CSS(가령 버튼이 눌렸을 때의 CSS, 에러가 발생했을 때의 버튼 css, display: none; 등)
    THEME CSS
    특정한 클래스별로 테마를 별도로 줘야할 때(폰트나 백그라운드의 색상 등)

  • 주의 사항

    • id 셀렉터는 되도록 사용하지 말것. js에서 쓰는 케이스가 많기 때문에 아껴두기, 또 셀렉터간의 우선 순위가 많이 꼬입니다. 나중에 선언한 CSS가 묻히는 경우도 있습니다.
    • 생략가능한 셀렉터는 생략하여 코드를 간결하게 합시다.
    • CSS 를 시작할 때는 문자 깨짐을 방지하기 위해 @charset “utf-8”; 코드를 문서 최상단에 넣습니다.
    • !important는 사용하지 말것. 나중에 선언한 CSS 가 100% 묻힙니다. 문제를 해결하지 못했다는 의미이기 때문에 결국 연달아 사용하게 되어 버립니다.
    • 클래스명은 숫자로 시작하지말 것, 영어로 쓸 것. 대쉬(-)를 제외한 특수기호도 제외합시다. HTML은 원래 대소문자를 구별하지 않지만 가끔 특수한 상황에서 브라우저가 대소문자를 구별할 때가 있기 때문에 카멜케이스 표기법은 사용하지 않습니다. 모듈명을 우선으로 할지 의미를 우선으로 할지는 본인의 선택입니다. 섞어서 쓰는 걸 권장합니다.(모듈 + 의미)
    • 요소셀렉터는 특정요소의 자식요소에만 사용합니다.(단독 혹은 셀렉터의 첫번째 순서로 사용하지 마세요). 요소가 어디서 몇번이나 어느자식으로 쓰일 지 모르기 때문입니다.(예기치 못한 변수를 예방하는 차원에서) 
                  
              header {}  —> X
              header li {} —> X
              .head_part header li {} —> O
